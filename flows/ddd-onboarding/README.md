# Onboarding System - Telegram Bot Integration

## Что это такое? (What is it?)

**Onboarding System** — это система ввода пользователей в приложение с интеграцией Telegram бота для получения подписки.

                                                 │ подписку
                                                 ▼

## Сценарии использования

### 📱 Сценарий 1: Публичное приложение (обязательный онбординг)

**Конфигурация:**
```env
SUBSCRIPTION_URL_MAIN=          # Не настроено
SHOW_ONBOARDING=true            # Показывать
```

**Что происходит:**
```
1. Пользователь запускает приложение
         │
         ▼
2. Видит экран "Welcome" с Telegram ботом
         │
         ▼
3. Нажимает "Go to Telegram Bot"
         │
         ▼
4. Открывается Telegram → Бот выдает подписку
         │
         ▼
5. Пользователь возвращается в приложение (deep link)
         │
         ▼
6. Видит экран "Settings Received"
         │
         ▼
7. Нажимает "Get Started"
         │
         ▼
8. Попадает на главный экран
```

**Результат**: ✅ Пользователь получил подписку и подключился к VPN

---

### 🏢 Сценарий 2: Корпоративное приложение (пропуск онбординга)

**Конфигурация:**
```env
SUBSCRIPTION_URL_MAIN=https://corp.vpn/token  # Настроено
SHOW_ONBOARDING=false                         # Не показывать
```

**Что происходит:**
```
1. Пользователь запускает приложение
         │
         ▼
2. Сразу видит главный экран (онбординг пропущен)
         │
         ▼
3. Подключается к VPN
```

**Результат**: ✅ Сотрудник сразу использует VPN без лишних шагов

---

### 🎯 Сценарий 3: Гибкое приложение (опциональный онбординг)

**Конфигурация:**
```env
SUBSCRIPTION_URL_MAIN=https://vpn.service/token  # Настроено
SHOW_ONBOARDING=true                             # Показать, но можно пропустить
```

**Что происходит:**
```
1. Пользователь запускает приложение
         │
         ▼
2. Видит экран "Welcome"
         │
         ├─────────────┐
         │             │
         ▼             ▼
    [Skip]        [Go to Bot]
         │             │
         │             │ (получить доп. подписку)
         │             │
         ▼             ▼
3. Главный экран ←─── Шаг 2
```

**Результат**: ✅ Пользователь может пропустить или получить дополнительную подписку

## Технические детали

### Deep Link схема

**Схема**: `vpnclient://`

**Как работает:**
```
Telegram Bot
     │
     │  Пользователь нажимает "Вернуться в приложение"
     │
     ▼
Открывается ссылка: vpnclient://
     │
     │  Приложение перехватывает ссылку
     │
     ▼
DeepLinkService.handleDeepLink("vpnclient://")
     │
     │  Проверяет: онбординг не завершен?
     │
     ▼
OnboardingService.setCurrentStep(1)
     │
     │  Переключение на Шаг 2 ("Settings Received")
     │
     ▼
Пользователь видит экран успеха
```

### Хранение состояния

```dart
// Используемые ключи в SharedPreferences
_onboardingCompletedKey      // Завершен ли онбординг
_onboardingSkippedKey        // Был ли пропущен
_lastOnboardingVersionKey    // Версия онбординга
_currentOnboardingStepKey    // Текущий шаг (0 или 1)
```

**Пример:**
```dart
// После завершения онбординга
prefs.setBool('vpnclient_onboarding_completed', true);
prefs.setString('vpnclient_last_onboarding_version', '1.0.0');

// При следующем запуске
bool completed = prefs.getBool('vpnclient_onboarding_completed') ?? false;
// completed = true → показать главный экран
```

## Troubleshooting

### ❌ Онбординг не показывается

**Проверьте**:
1. `SUBSCRIPTION_URL_MAIN` настроен?
2. `SHOW_ONBOARDING=false`?
3. Онбординг уже был завершен ранее?

**Решение**:
```bash
# Сбросить состояние онбординга (для тестов)
adb shell pm clear com.example.vpnclient  # Android
# или использовать в коде:
await OnboardingService().resetOnboarding();
```

---

### ❌ Deep link не работает

**Проверьте**:
1. `ENABLE_DEEP_LINKS=true` в `.env`
2. Android: intent filters в AndroidManifest.xml
3. iOS: URL types в Info.plist

**Пример AndroidManifest.xml**:
```xml
<intent-filter>
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="vpnclient" />
</intent-filter>
```

---

### ❌ Telegram бот не открывается

**Проверьте**:
1. `TELEGRAM_BOT_URL` правильный формат? (`t.me/username`)
2. Telegram установлен на устройстве?

**Решение**:
- Приложение пытается открыть через приложение Telegram
- Если не установлено → открывает веб-версию

### Добавление новых шагов

```dart
List<OnboardingStep> _getSteps() {
  return [
    // Шаг 1: Welcome
    OnboardingStep(
      title: 'Welcome',
      // ...
    ),
    
    // Шаг 2: Разрешения (НОВЫЙ)
    OnboardingStep(
      title: 'Permissions',
      description: 'Allow VPN connection',
      icon: Icons.shield,
      color: const Color(0xFF2196F3),
    ),
    
    // Шаг 3: Выбор сервера (НОВЫЙ)

      title: 'Choose Server',
      description: 'Select your preferred location',
      icon: Icons.public,
      color: const Color(0xFF4CAF50),
    ),
    
    // Шаг 4: Success
    OnboardingStep(
      title: 'Settings Received',
      // ...
      isLast: true,
    ),
  ];
}
```
